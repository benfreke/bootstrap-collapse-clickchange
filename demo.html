Collapse inside a collapse

This sort of structure is useful for categorising FAQs. I'm sure there are many other uses as well.

These icons do not change, to highlight that each collapse grouping has it's own attributes. For this to work as expected, clickchange must be initialised on the sub collapse group as well.

Nifty. In a future diversion the requirement to initialise clickchange will be removed.

Demo of a stand alone collapse


This button code is called manually.

This is the code used on the button: 

<button id="individual" type="button" class="btn btn-primary" data-toggle="collapse" data-target="#demo-single">
    <span class="toggleonclick glyphicon glyphicon-thumbs-up hidden"></span>
    <span class="toggleonclick">Click me please!</span>
    <span class="toggleonclick hidden">Thank you for clicking!</span>
</button>

This is the code used to initialise clickchange. 

$('#individual').clickChange({
    when: 'before',
    parentclass: 'btn-primary btn-success',
    iconchange: true,
    iconprefixadd: false,
    iconprefix: 'toggleonclick',
    iconclass: 'hidden'
});

Instead of changing icons, it simply toggles the hidden class, changing what is displaying to the user.
